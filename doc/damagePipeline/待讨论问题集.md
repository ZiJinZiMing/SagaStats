# Part A: 待实施

所有设计决策已确认。按依赖关系排序。

---

## A1. Execute 返回 Fact，PipelineAsset 负责写入 DC ✅ 已决策

**决策**：DPU 的 Execute 是纯函数——读 DC（只读）、执行逻辑、返回 Fact（FInstancedStruct）。PipelineAsset 拿到返回值后以 DPU 身份标识写入 DC。DPU 不负责写入 DC。

**影响范围**：
- `UDPULogicBase::Execute` 签名从 `void Execute(DC)` 改为返回 `FInstancedStruct`
- `UPipelineAsset::Execute` 中调用 Logic 后负责 `DC->SetFact(DPUName, ReturnedFact)`
- 所有 DPU Logic 子类（DPUTestLogics.h）重写为返回 Fact
- TestActor 无需改动（调用的是 PipelineAsset::Execute）

**R2 自洽性**：✅ DPU 完全不知道自己的身份，不知道写入索引。

## A2. ConditionNode 引用 DPUDefinition 而非 FName ✅ 方向明确

**方向**：ConditionNode_FactQuery 和 Compare 的属性从 `FName FactKey` 改为 `TObjectPtr<UDPUDefinition> SourceDPU`。

**影响范围**：
- ConditionNode_FactQuery / Compare 属性类型变更
- GetConsumedFacts() → GetConsumedDPUs()，返回 `TArray<FName>`（从 SourceDPU->DPUName 取）
- GetOptions 下拉改为 Content Browser 选择 DataAsset
- 拓扑排序匹配逻辑从 Fact Key 改为 DPU Name
- A3/A4 自然跟随解决

## A3. DPUDefinition.Produces 从 TArray\<FName\> 改为 UScriptStruct* ✅ 模型已确认

**方向**：每个 DPU 产出一个 Fact 类型。和 A2 同步实施。

## A4. 去掉 FactKeyToStruct 映射表 ✅ A2 完成后自然解决

**方向**：ConditionNode 引用 DPUDefinition 后，联动下拉直接从 `SourceDPU->Produces` 获取类型。FactMethodRegistry 中的 FactKeyToStruct/GetRegisteredFactKeys/GetMethodsForFactKey 可删除。

## A5. 领域方法注册方式 ✅ 已决策（实现层自行决定）

**决策**：这是纯实现层问题，不影响模型。

**推荐方向**：
- C++ 侧：注册宏（一行代码注册一个方法）或保持当前手动 lambda
- 蓝图侧：UFactDefinition DataAsset + UAssetManager 自动发现 + IAssetRegistry::OnAssetAdded() 编辑器实时监听

**注意**：FInstancedStruct 在 UE 5.5+ 已稳定，5.4 有已确认的序列化 bug。

## A6. 表现选取阶段

模型规范已定义 Channel、Priority、CombinedPresentation、选取算法。代码中 BasePriority 声明但未使用。**当前最大的模型-代码差距。**

**依赖**：无，可独立实施。

## A7. 运行时 Produces 验证

DPU 可写入未声明字段，代码无检查。Debug 模式下加执行前后 diff DC 验证。

**依赖**：无，可独立实施。

# Part C: 已解决归档 🟢

---

## C1. Condition 表达力边界

**消解**：v4.5 条件树 + 领域方法。Condition 只调用方法，方法复杂度由 Fact 开发者控制。R4 不受影响。原先"两层 Condition"方案被自然替代。

## C2. StringCondition Usability

**消解**：已删除，替代为条件树 + GetOptions 下拉。

## C3. Produces 独占性

**消解**：v4.5 每个 DPU 以自身身份为索引，不存在共享索引。互斥 DPU 消费者用 OR。

## C4. Pipeline 执行架构

**消解**：PipelineManager 已删除。PipelineAsset 自洽。

## C5. 独立 Fact Key 概念

**消解**：经三步检验确认为偶然复杂度。v4.5 以 DPU 身份标识替代。

## C6. Execute 写入方式决策

**决策**：方案 A（返回 Fact）。见讨论总结决策 2。

## C7. 领域方法注册方式决策

**决策**：纯实现层问题，Claude Code 自行决定。见讨论总结决策 4。
